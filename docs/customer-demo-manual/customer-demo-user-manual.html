Section 1 · Brand & Disclosures

1. Brand & Disclosures الهوية والإفصاحات

EN

Taāzur (تآزر, "mutual support") is the default brand for the customer demo. The platform is presented as powered by UDS-Noon JV. Both strings are centrally configurable per tenant through packages/config/src/brand.ts; nothing in the application code is hard-coded with the brand name. Logos, lock-ups, and bilingual typography are produced from SVG sources under apps/web/public/brand/.

AR

اسم تآزر هو العلامة الافتراضية للنسخة التجريبية. تُعرض المنصة على أنها مدعومة من UDS-Noon JV. كلتا العبارتين قابلتان للتكوين لكل عميل من خلال packages/config/src/brand.ts؛ ولا توجد قيمة ثابتة للعلامة داخل كود التطبيق. الشعار، التركيبات البصرية، والطباعة ثنائية اللغة تُولَّد من ملفات SVG داخل apps/web/public/brand/.

Mandatory disclosures · إفصاحات إلزامية

Fictional data only · بيانات وهمية

The Rukn Energy Services tenant shown in this demo is fictional. No record represents a real person, a real employer, or a real government transaction. The CR number, GOSI number, VAT number, Iqama numbers, IBANs, and employee IDs are demonstrably fake.

Mock integrations · تكاملات محاكاة

Qiwa, Mudad/WPS, GOSI, Muqeem, ZATCA, the bank payroll file, and the Microsoft Teams connectors are deterministic simulations. No call is ever placed to a real authority system. Production rollout requires separately provisioned credentials and vendor onboarding.

Operational Demo mode · وضع التشغيل التجريبي

All demo identities and the company-onboarding wizard are gated behind DEMO_MODE=true. The four demo role buttons appear on the login screen only when this flag is set.

Section 2 · Demo Personas & RBAC

2. Demo Personas & RBAC الشخصيات التجريبية والصلاحيات

The customer demo ships four distinct testing personas. Each persona is a real NextAuth session with a different role, capability set, navigation, and seed employee record. The capability matrix is enforced at three layers: sidebar.tsx (UI), middleware.ts (route), and trpc/server.ts::protectedProcedure (procedure/data).

1. HR Manager admin

Persona: Reem Al-Harbi · People & Culture Director

All capabilities. Owns company onboarding, compliance profile, payroll runs, final settlements, and authority submissions. Single-tenant superuser.

Sees: every workspace, every page, every procedure.

2. HR Specialist specialist

Persona: Aisha Al-Otaibi · Payroll & GOSI Specialist

Operations-focused. Can run payroll, manage documents, approve leave, reconcile GOSI, submit Mudad files. Cannot edit company setup or final settlements.

Sees: company payroll, leave, documents, government.

3. Department Manager manager

Persona: Fahad Al-Qahtani · Eastern Operations Manager

Team-scoped. Approves leave, manages goals, runs performance reviews for direct reports, reviews engagement, submits expenses. Cannot view company-wide payroll.

Sees: team people, performance, engagement, time.

4. Employee employee

Persona: Omar Al-Dossary · Field Engineer

Self-service only. Sees own profile, requests leave, submits expenses, accesses personal documents and payslips. Cannot view other employees, company payroll, or admin settings.

Sees: profile, leave, expenses, personal documents.

Section 4 · HR Specialist Command Center

4. HR Specialist Command Center مركز قيادة أخصائي الموارد البشرية

What the HR Specialist does here

  • Run payroll, validate Mudad wage files, reconcile GOSI contributions.
  • Approve leave, manage employee documents, sign certificates.
  • Cannot edit company setup, billing, or final settlements.
  • Has access to the government integrations center (mock).

ما يقوم به أخصائي الموارد البشرية

  • تشغيل كشوف الرواتب، التحقق من ملفات حماية الأجور، ومطابقة اشتراكات التأمينات.
  • اعتماد الإجازات وإدارة مستندات الموظفين.
  • لا يستطيع تعديل إعدادات الشركة أو الفواتير أو التسويات النهائية.
  • يملك وصولاً إلى مركز التكاملات الحكومية (محاكاة).

RBAC enforcement · تطبيق الصلاحيات

The HR Specialist role lacks compliance:manage and settings:manage. The capability-filtered sidebar removes the Nitaqat, Settings, and Company Onboarding links automatically. Server-side middleware and tRPC procedures also enforce the same rule on direct URL access.

Section 6 · Employee Self-Service

6. Employee Self-Service الخدمة الذاتية للموظف

What the Employee does here

  • See their own command center, profile, payslips, and assets.
  • Request leave, view balances, cancel a personal request.
  • Submit travel & expense claims, attach receipt details, track approval.
  • Enroll in learning courses, mark skills verified.
  • Attempting to open /payroll redirects to /?access=denied.

ما يقوم به الموظف

  • عرض لوحة الخدمة الذاتية الخاصة به، وملفه الشخصي، وكشوف الرواتب، والأصول المسلَّمة.
  • طلب إجازات، وعرض الأرصدة، وإلغاء الطلبات الشخصية.
  • تقديم مطالبات السفر والمصروفات، وإرفاق الإيصالات، ومتابعة الاعتماد.
  • التسجيل في دورات التعلم وتأكيد المهارات.
  • محاولة فتح /payroll تعيد التوجيه إلى /?access=denied.

Ownership scoping · نطاق الملكية

All employee-facing queries are filtered by employeeId extracted from the JWT. The Employee persona can never see another employee's record, company payroll, or admin pages.

Section 10 · Mock vs Live disclosure

10. Mock vs Live Disclosure إفصاح: محاكاة مقابل الإنتاج

The customer demo uses deterministic mocks for every authority and external-provider integration. This is by design — the demo runs entirely in the browser and a Next.js edge runtime, with no live credentials, no API keys, and no outbound calls.

What "MOCK" means

  • Clicking "Run Qiwa contract check" produces a deterministic mock response; no call is made to the real Qiwa API.
  • Submitting a Mudad wage file triggers a deterministic validation; the file is not uploaded anywhere.
  • GOSI reconciliation produces illustrative output; no real GOSI subscription is touched.
  • Bank payroll file generation produces a downloadable PDF only; the bank is not contacted.
  • Microsoft Teams webhook receives a local simulated delivery.

تستخدم النسخة التجريبية محاكاة حتمية لكل تكامل مع الجهات الخارجية أو المزودين. هذا مقصود — النسخة تعمل بالكامل داخل المتصفح وبيئة Next.js، دون بيانات اعتماد حية أو مفاتيح API أو أي اتصال صادر.

معنى "محاكاة"

  • النقر على "Run Qiwa contract check" ينتج استجابة محاكاة حتمية؛ لا يتم الاتصال بـ API قوى.
  • تقديم ملف حماية الأجور يؤدي إلى تحقق محاكاة؛ لا يُرفع الملف إلى أي مكان.
  • مطابقة التأمينات تنتج نتائج توضيحية فقط.
  • توليد ملف رواتب البنك ينتج ملف PDF للتنزيل فقط.

Production rollout checklist · قائمة نشر الإنتاج

  1. Obtain production credentials from Qiwa, Mudad, GOSI, Muqeem, ZATCA, and the payroll bank.
  2. Complete Saudi legal validation of all EOSB, GOSI, leave, and overtime rules.
  3. Run schema-per-tenant migration against the production PostgreSQL instance.
  4. Apply RLS policies and verify tenant isolation tests on the production database.
  5. Replace mock adapters with provider-specific adapters behind the same interface.
  6. Configure tenant brand profile (logo, lock-up, color tokens, locale).
  7. Onboard HR Manager, HR Specialist, Department Managers, and employees.
  8. Run first payroll in shadow mode; reconcile with bank file; promote to live.
  9. Activate Nitaqat posture after first Qiwa submission.
  10. Schedule first engagement pulse; gather first 30-day onboarding feedback.
